# PRD：多门店移动端商城/点餐系统

## 一、项目背景

本项目旨在为多门店场景（如连锁餐饮、零售）提供一套高效、易用的移动端商城/点餐解决方案。用户可通过手机端访问各门店，浏览商品、下单、管理个人信息，门店可灵活配置菜单、营业状态等。

## 二、核心功能

### 1. 门店管理
- 支持多门店切换，门店信息展示（名称、状态、公告等）
- 门店营业状态实时显示（营业中/休息中/已关闭）
- 门店菜单独立配置

### 2. 商品与菜单
- 商品分组、分类展示
- 支持商品图片、价格、描述等信息
- 菜单数据支持服务端动态拉取，支持缓存与 SSR

### 3. 购物车与下单
- 商品加入购物车
- 支持下单、订单确认
- 订单数据与门店、用户关联

### 4. 用户体系
- 微信授权登录、开发环境快捷登录
- 用户信息管理
- 用户联系人管理

### 5. 订单与业绩
- 订单数据管理
- 业绩统计与日志记录

### 6. 性能与体验
- 支持 SSR、服务端渲染
- 性能埋点与日志
- 响应式布局，适配主流移动端

## 三、技术架构

- 前端框架：Nuxt3（Vue3）
- UI 组件库：Vant
- 状态管理：Pinia
- 样式工具：UnoCSS
- 后端接口：基于 Nuxt3 Server API，RESTful 风格
- 数据库：MySQL（通过 mysql2 连接）
- 认证：微信 OAuth、开发环境快捷登录
- 依赖管理：pnpm

## 四、接口设计（示例）

- `/api/menu/[storeCode]`：获取指定门店菜单
- `/api/stores/[storeCode]/info`：获取门店信息
- `/api/stores/[storeCode]/tables`：获取门店桌台信息
- `/api/users/[userId]/contacts`：用户联系人管理
- `/api/auth/wechat-auth-url`：获取微信授权链接
- `/api/auth/dev-login`：开发环境登录
- `/api/performance/log`：性能埋点日志

## 五、非功能性需求

- 支持 Docker 部署
- 代码规范遵循 Vue3 最佳实践
- 组件、页面、接口分层清晰，易于扩展
- 支持多环境配置
